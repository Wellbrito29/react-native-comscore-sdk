REGRAS DE CONFORMIDADE COMSCORE

Siga estritamente estas regras:
1. Não use APIs não documentadas da Comscore.
2. Não invente classes, métodos, labels ou fluxos.
3. publisherId é obrigatório.
4. applicationName é opcional.
5. usagePropertiesAutoUpdateMode e usagePropertiesAutoUpdateInterval são opcionais.
6. O section tracking deve usar notifyViewEvent com o label ns_category.
7. Para telas gerais como home ou splash, ns_category deve aceitar string vazia.
8. O consentimento manual deve usar cs_ucfr como Persistent Label.
9. Os únicos valores válidos para cs_ucfr são:
   - "0"
   - "1"
   - ""
10. Se o consentimento mudar depois do start, a implementação deve:
   - atualizar cs_ucfr
   - disparar hidden event
11. child-directed mode deve ser aplicado antes do start.
12. implementation validation mode deve ser aplicado antes do start.
13. validation mode deve ficar restrito a debug/validação e nunca permanecer habilitado em produção.
14. notifyUxActive() e notifyUxInactive() só devem ser usados em cenários reais de UX em background, como playback.

DETALHES OBRIGATÓRIOS — API PÚBLICA

Implemente uma API TypeScript equivalente a esta, adaptando ao template existente:

type ComscoreConsentValue = "0" | "1" | ""

type ComscoreAutoUpdateMode =
  | "foregroundOnly"
  | "foregroundAndBackground"
  | "disabled"

type ComscoreConfig = {
  publisherId: string
  applicationName?: string
  autoUpdateMode?: ComscoreAutoUpdateMode
  autoUpdateIntervalSeconds?: number
  childDirected?: boolean
  validationMode?: boolean
  startOnlyWhenUIIsVisible?: boolean
  initialConsent?: ComscoreConsentValue
  debugLogs?: boolean
}

interface ComscoreModule {
  initialize(config: ComscoreConfig): Promise<void>
  notifyUxActive(): Promise<void>
  notifyUxInactive(): Promise<void>
  trackSection(sectionName?: string | null): Promise<void>
  updateConsent(value: ComscoreConsentValue): Promise<void>
  setChildDirected(enabled: boolean): Promise<void>
  setValidationMode(enabled: boolean): Promise<void>
  getVersion(): Promise<string>
}

Regras:
- initialize deve validar publisherId
- initialize deve ser idempotente
- trackSection deve aceitar string vazia
- updateConsent deve funcionar antes e depois do start
- setValidationMode deve falhar de forma controlada ou virar no-op em release
- tipos estritos
- mensagens de erro claras
